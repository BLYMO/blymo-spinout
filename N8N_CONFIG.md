# n8n Tenant Configuration Reference

This document covers all environment variables injected into each tenant's ECS task definition via the `modules/tenant/main.tf` file. These control n8n's behaviour, security, email, and billing limits on a per-tenant basis.

---

## 🗄️ Database

| Variable | Value | Notes |
|---|---|---|
| `DB_TYPE` | `postgresdb` | Tells n8n to use PostgreSQL |
| `DB_POSTGRESDB_HOST` | `var.db_host` | Shared RDS instance hostname |
| `DB_POSTGRESDB_PORT` | `var.db_port` | Default: `5432` |
| `DB_POSTGRESDB_DATABASE` | `postgres` | Root database name |
| `DB_POSTGRESDB_SCHEMA` | `var.db_schema` | Tenant-specific schema — this is the isolation layer |
| `DB_POSTGRESDB_USER` | Secret (Secrets Manager) | Pulled from shared RDS credentials secret |
| `DB_POSTGRESDB_PASSWORD` | Secret (Secrets Manager) | Pulled from shared RDS credentials secret |

> **Note:** `DB_POSTGRESDB_SCHEMA` is the critical variable for multi-tenant isolation. Each tenant's data is scoped to its own Postgres schema. Without this, all tenants would share the `public` schema.

## 🌐 URL & Routing

| Variable | Value | Notes |
|---|---|---|
| `N8N_HOST` | `{subdomain}.trybase.io` | The public hostname the instance is served on |
| `N8N_PROTOCOL` | `https` | Forces HTTPS everywhere |
| `N8N_PORT` | `5678` | Internal container port |
| `WEBHOOK_URL` | `https://{subdomain}.trybase.io` | Base URL for webhook triggers — must match public URL |
| `N8N_EDITOR_BASE_URL` | `https://{subdomain}.trybase.io` | Used for links inside the editor UI |

> **Note:** `WEBHOOK_URL` is critical. Without it, n8n generates incorrect webhook URLs that won't work from the public internet.

---

## 🔐 Security

| Variable | Value | Notes |
|---|---|---|
| `N8N_ENCRYPTION_KEY` | Secret (Secrets Manager) | **Per-tenant**, auto-generated 32-char key. Encrypts all stored credentials (API keys, passwords) at rest. |

> **Critical:** Every tenant must have a **unique** encryption key. If keys are shared, a compromised tenant could decrypt another's stored credentials. Keys are auto-generated by Terraform's `random_password` resource and stored in Secrets Manager at `n8n-hosting/tenant/{tenant_id}/encryption-key`.

## ⚡ Execution Limits (Billing Protection)

| Variable | Value | Notes |
|---|---|---|
| `EXECUTIONS_TIMEOUT` | `3600` | Max execution duration in seconds (1 hour) |
| `EXECUTIONS_TIMEOUT_MAX` | `7200` | Hard ceiling — users cannot override above this (2 hours) |
| `EXECUTIONS_DATA_MAX_AGE` | `7` | Auto-delete execution logs older than 7 days — prevents DB bloat |
| `EXECUTIONS_DATA_SAVE_ON_SUCCESS` | `all` | Save logs of successful runs |
| `EXECUTIONS_DATA_SAVE_ON_ERROR` | `all` | Save logs of failed runs |
| `N8N_CONCURRENCY_PRODUCTION_LIMIT` | `5` | Max parallel workflow executions per instance |

> **Tip for pricing tiers:** Increase `EXECUTIONS_TIMEOUT` and `N8N_CONCURRENCY_PRODUCTION_LIMIT` for Pro and Business plans to justify the price difference.

## 📧 SMTP Email (via Resend)

| Variable | Value | Notes |
|---|---|---|
| `N8N_EMAIL_MODE` | `smtp` | Enables SMTP email sending |
| `N8N_SMTP_HOST` | `smtp.resend.com` | Resend SMTP endpoint |
| `N8N_SMTP_PORT` | `465` | SSL port |
| `N8N_SMTP_SSL` | `true` | Use SSL |
| `N8N_SMTP_USER` | `resend` | Resend requires this literal username |
| `N8N_SMTP_PASS` | Secret (Secrets Manager) | Resend API key stored at `n8n-hosting/resend-smtp-api-key` |
| `N8N_SMTP_SENDER` | `noreply@{subdomain}.trybase.io` | Per-tenant sender identity |

> **Note:** Since `trybase.io` is a verified Resend domain, all subdomains inherit verification automatically. The sender address changes per tenant, giving each instance its own email identity.
>
> **Business plan upgrade path:** Replace sender with `noreply@{custom_domain}` using the tenant's own verified domain as a premium feature.

## 🤖 AI Assistant

| Variable | Value | Notes |
|---|---|---|
| `N8N_AI_ASSISTANT_BASE_URL` | `https://ai-assistant.n8n.io` | Enables built-in AI chat in the workflow editor |

> **Current:** Pointing to n8n's hosted service.
>
> **Future:** Build a proxy at `https://ai.trybase.io` that:
> 1. Intercepts requests from n8n editor
> 2. Forwards to OpenAI/Claude with your own API key
> 3. Returns responses in n8n's expected format
>
> This enables per-plan AI access, custom model selection (GPT-4o, Claude, etc.), and removes data dependency on n8n's servers. Change one line to activate: `N8N_AI_ASSISTANT_BASE_URL = "https://ai.trybase.io"`.
>
> To discover the API contract, deploy a logging proxy and watch the request/response shapes in real usage.

## 🎨 Privacy & UX

| Variable | Value | Notes |
|---|---|---|
| `N8N_DIAGNOSTICS_ENABLED` | `false` | Stops n8n sending usage/error data to their Sentry + analytics |
| `N8N_VERSION_NOTIFICATIONS_ENABLED` | `false` | Hides "A new version is available" banner (you manage upgrades) |
| `N8N_PERSONALIZATION_ENABLED` | `false` | Skips the first-login questionnaire — users land straight in the editor |
| `N8N_TEMPLATES_ENABLED` | `true` | Keeps the template library accessible — useful for new users |

## 🗺️ Adding New Tenants

New tenants are provisioned via the Step Function → CodeBuild pipeline, which dynamically writes a `tenant_{id}.tf` file and runs `terraform apply -target=module.{tenant_id}`.

All variables above are automatically injected using the `tenant_id` and `subdomain` values from the Step Function input payload.

For **manual testing**, add a new module block to `tenants.tf`:

```hcl
# Generate a unique encryption key
resource "random_password" "miles_encryption_key" {
  length  = 32
  special = false
}

resource "aws_secretsmanager_secret" "miles_encryption_key" {
  name                    = "n8n-hosting/tenant/miles/encryption-key"
  recovery_window_in_days = 0
}

resource "aws_secretsmanager_secret_version" "miles_encryption_key" {
  secret_id     = aws_secretsmanager_secret.miles_encryption_key.id
  secret_string = random_password.miles_encryption_key.result
}

module "miles-davis" {
  source    = "./modules/tenant"
  tenant_id = "miles-davis"
  subdomain = "miles-davis"
  db_schema = "miles_davis" # Use underscores for the DB schema name!

  vpc_id                         = module.vpc.vpc_id
  private_subnet_ids             = module.vpc.private_subnets
  ecs_cluster_name               = aws_ecs_cluster.main.name
  alb_listener_arn               = aws_lb_listener.https.arn
  alb_security_group_id          = aws_security_group.alb.id
  db_host                        = aws_db_instance.main.address
  db_port                        = aws_db_instance.main.port
  db_credentials_secret_arn      = aws_secretsmanager_secret.db_credentials.arn
  vpc_endpoint_security_group_id = aws_security_group.vpc_endpoints.id
  n8n_encryption_key_secret_arn  = aws_secretsmanager_secret_version.miles_encryption_key.arn
  smtp_api_key_secret_arn        = aws_secretsmanager_secret.smtp_api_key.arn
  alb_listener_rule_priority     = 101 # Must be unique!
}
```
